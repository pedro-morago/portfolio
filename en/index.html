About$ cat about.md

I've been a QA Engineer since 2021, focused on software quality end to end: defining quality strategy and designing test cases across the whole testing pyramid, combining manual and automated testing. I work daily with Playwright, Cypress and TypeScript for E2E, and with Java, REST Assured and JUnit 5 for API automation, always with tests integrated into CI/CD pipelines and within fully remote, distributed agile teams.

Before software, I completed a BSc in Mathematics at the University of Cantabria. More than a syllabus, the degree trains a way of working: formulating problems in precise terms, paying attention to detail, following complex reasoning, and constructing logical arguments and exposing flawed ones. That training maps naturally onto QA: testing software is, in essence, searching for counterexamples to the claim that it works.

That foundation also applies directly: at IriusRisk I designed a mixed-integer linear programming model to parallelize E2E test execution in CI/CD, cutting pipeline validation time by around 40 % with no extra infrastructure cost. That idea led to my personal project CI Shard Advisor.

Beyond finding bugs, I'm interested in building processes and tools that help keep them out of production. That's why my personal projects revolve around testing and automation, and I write about QA in my QA Wiki (in Spanish), where I practice another skill the degree also trains: communicating complex technical information clearly. ISTQB CTFL v4.0 certified.

CI Shard Advisor

published

Optimal parallelization of Cypress suites in CI, without relying on Cypress Cloud.

A tool that analyzes your Cypress run reports and recommends the optimal sharding strategy for your pipeline: it measures current performance, detects imbalances between containers and suggests the exact spec partition that minimizes total execution time, with optimality certification via a branch & bound algorithm.

  • 100 % local, private analysis: reports never leave the browser.
  • Three interfaces: interactive web app, CLI with a quality-gate mode, and a local REST API.
  • Actionable recommendations: per-container spec lists ready to paste into GitHub Actions or Bitbucket Pipelines.
  • Pure core with hexagonal architecture, tested with property-based, metamorphic and invariant tests.

Screen Recorder · QA mode

published

Chrome extension for reporting bugs with full evidence in a single recording.

A browser extension (Manifest V3) that records your screen, window or tab while automatically capturing everything a developer needs to reproduce the bug: console logs synced with the video, uncaught exceptions, network requests as HAR, and numbered user steps. No time limits, no watermark, no account.

  • QA mode: console, errors, 404s and HAR synced to the video by timestamp.
  • Live on-video annotations and keyboard-shortcut markers to flag the exact moment of the bug.
  • Automatic step recording (clicks, forms) exportable as *.steps.md, and typed values are never logged.
  • Generates the full report and files the issue directly in Jira Cloud or Linear.
  • Zero external dependencies: native browser APIs only.
